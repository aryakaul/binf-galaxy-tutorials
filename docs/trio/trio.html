1.

This tutorial will walk you through how to perform variant calling on a family of individuals on Galaxy's platform. To start with we will make use of pre-aligned BAM reads from the father, mother, and son. Find these in the following data library found here. Select all of these files and click on 'Export to History' as datasets. Create a new history to send the files to. 
See image hint

2.

Now rename these files to their corresponding individuals. NA12877 corresponds to the son, NA12890 corresponds to the mother, and NA12889 corresponds to the father. Select all the bam files corresponding to a certain individual and select build dataset list. Name them something corresponding to the individual it came from and select 'Hide Original Elements' to tidy up the history. Click 'Create List'.
See image hint

3.

We will now generate a VCF file containing variants for each individual in this family using the tool 'FreeBayes'. To use this tool, navigate to the left tool bar and search for it. Click on it and change the following options: 1. Set 'Merge Output VCFs' 2. Use one of the BAM dataset collections created in Step 2 3. Set hg38 as the reference genome 4. Choose Simple diploid calling with filtering and coverage and require at least 10 reads to cover to process a given site. Execute this job and repeat for each BAM collection. You should end up with 3 VCFs.
See image hint

4.

We will now compare the VCFs from each individual. To do this, we will make use of the "VCF-VCFintersect" tool available through Galaxy. Select each possible pairwise combination of the VCFs and choose hg38 as the reference genome and run each. Leave the rest of the options unchanged.
